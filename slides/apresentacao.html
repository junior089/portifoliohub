Entrega final

PortfolioHUB + IA Gemini

Implantação de uma plataforma de portfólio digital com GitHub, segurança, documentação, testes e apresentação final.

Carlos Alberto Junior2026

Objetivo

Centralizar projetos e evidências

  • Exibir projetos acadêmicos e pessoais.
  • Registrar o processo de implantação.
  • Usar GitHub como ambiente de versionamento e compartilhamento.
  • Utilizar Gemini como apoio na tomada de decisão técnica.

Planejamento

Trilha de implantação

1. Preparar

Clone, estrutura de pastas e revisão do escopo.

2. Integrar

GitHub API, documentação e cards de projeto.

3. Validar

Segurança, testes, relatório e apresentação.

GitHub

Versionamento e integração

  • Repositório dedicado ao projeto.
  • Histórico de commits e tags de versão.
  • GitHub Pages como ambiente de publicação.
  • GitHub REST API usada para metadados públicos do repositório.

Segurança

Controle de acesso e proteção

IAM

Owner, collaborator e reader com menor privilégio.

Branch

Recomendação de Pull Request e proteção da main.

CSP

Política de conteúdo restritiva sem scripts inline.

Gemini

IA como apoio, não como atalho

O Google Gemini foi utilizado de forma ativa para planejar a implantação, sugerir políticas de segurança (CSP, Branch Protection) e estruturar a matriz de testes. Nenhuma chave de API ou credencial sensível foi inserida no frontend, priorizando a segurança por design.

Testes

Validações executadas

  • Navegação, tema, filtros e responsividade.
  • Consulta à API e fallback offline.
  • Links internos para documentação, slides e relatório.
  • Relatório final em PDF atualizado.

Conclusão

Resultado final

O PortfolioHUB ficou organizado como uma entrega completa, com implantação documentada, segurança descrita, integração com GitHub, material de apresentação e evidências para avaliação.
