Sobre

Um hub para projetos, documentação e presença profissional

O PortfolioHUB reúne projetos acadêmicos, projetos pessoais, documentação técnica e uma página pública preparada para GitHub Pages. A proposta é apresentar a evolução em desenvolvimento web, organização de repositórios e boas práticas de versionamento.

Segurança & IAM

Políticas de Segurança e Controle de Acesso

Políticas e regras de colaboração recomendadas por IA e auditadas para garantir a integridade do código e a segurança do deploy.

Controle de Acesso

Gestão IAM

Organização de permissões de usuários no GitHub divididas em papéis estratégicos para garantir o princípio do menor privilégio.

  • Owner: Acesso total às configurações e faturamento.
  • Collaborator: Permissão de escrita via Pull Requests.
  • Reader: Acesso de leitura para auditoria acadêmica.
Versionamento

Branch Protection

Regras configuradas no repositório GitHub para a branch main para impedir alterações acidentais e deploys inseguros.

  • Exigência de Pull Request com revisão dupla.
  • Assinatura de commits obrigatória.
  • Bloqueio de pushes diretos e force push.
Auditoria

Varredura & CSP

Auditorias sugeridas pelo Google Gemini para proteção ativa contra vulnerabilidades web (OWASP Top 10).

  • Content Security Policy (CSP) via Meta Tag.
  • Dependabot ativado para alerta de dependências.
  • Secret Scanning ativo contra vazamento de tokens.
