Gestão de usuários

Matriz de acesso para colaboração segura

Os papéis abaixo orientam como usuários interagem com o repositório, a documentação e a publicação do PortfolioHUB.

PapelPermissãoUso
OwnerAdminGerencia Pages, permissões e branch principal.
MaintainerWriteRevisa Pull Requests e mantém documentação.
CollaboratorBranch/PRContribui sem alterar diretamente a main.
AvaliadorReadConsulta site, relatório e evidências.

Git e GitHub

Compartilhamento e controle de acesso configurados

01

Repositório dedicado

Estrutura separada por site, assets, documentação, projetos, relatório e apresentação.

02

Fluxo por branch

Alterações relevantes devem nascer em branch própria, passar por commits e chegar por Pull Request.

03

Revisão antes do merge

O template de PR reforça testes, links, documentação e ausência de segredos no código.

04

Publicação estática

GitHub Pages permite compartilhar o PortfolioHUB sem expor backend, banco de dados ou credenciais.

Segurança

Controles aplicados e recomendados

A entrega combina proteção no front-end, documentação de IAM e práticas de repositório para reduzir risco de alteração indevida ou vazamento de informações.

Content Security Policy

Scripts e estilos limitados à própria origem, conexão externa restrita à API pública do GitHub.

Sem credenciais no código

O projeto não usa tokens, senhas, chaves de API ou dados sensíveis no front-end.

Proteção de branch

Recomendação para bloquear force push, exclusão da main e exigir Pull Request.

Fallback da API

A interface continua funcionando quando o limite público da GitHub API é atingido.
