Configuração inicial e integração com o repositório

Ambiente dedicado

O repositório junior089/portfoliohub-entrega-intermediaria foi utilizado como ambiente central da entrega. A estrutura separa página, assets, documentação, relatório, slides e projetos.

gh repo clone junior089/portfoliohub-entrega-intermediaria
cd portfoliohub-entrega-intermediaria
git checkout -b entrega-final

Integração no PortfolioHUB

A seção inicial consome a GitHub REST API para buscar metadados do repositório, com fallback local para não quebrar a interface em caso de limite de requisições ou acesso offline.

GET https://api.github.com/repos/junior089/portfoliohub-entrega-intermediaria

Segurança e IAM

Políticas de segurança e controle de acesso

As regras abaixo estão documentadas para aplicação no GitHub. O objetivo é reduzir risco de alteração indevida, vazamento de credenciais e execução de scripts não autorizados.

Acesso

Matriz IAM

  • Owner: administra repositório, Pages e permissões.
  • Collaborator: altera código por branch e Pull Request.
  • Reader/Avaliador: acessa documentação e evidências.
Git

Proteção de branch

  • Evitar push direto na main.
  • Usar Pull Request para revisão.
  • Bloquear force push e exclusão de branch protegida.
Web

CSP e segredos

  • Content Security Policy sem unsafe-inline.
  • Nenhum token, senha ou chave no código-fonte.
  • Uso de checklist para Dependabot e Secret Scanning.

Google Gemini

Apoio de IA no Planejamento e Segurança

O Google Gemini foi utilizado de forma ativa como consultor para planejar as etapas da implantação, formular as diretrizes de segurança no GitHub e orientar a estruturação do roteiro da entrega final.

Planejamento da Implantação

Uso de prompts voltados à engenharia de software para definir o cronograma de entrega, a divisão de branches e o fluxo de deploy no GitHub Pages.

Políticas de Segurança

Consultoria da IA para o estabelecimento de regras de Branch Protection no repositório GitHub e diretrizes de controle de acesso (IAM).
